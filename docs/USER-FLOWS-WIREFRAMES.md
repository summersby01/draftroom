# USER-FLOWS-WIREFRAMES

## 1. 주요 사용자 흐름

## Flow A. 로그인
1. 사용자가 로그인 페이지 진입
2. 이메일 또는 매직링크 로그인
3. 인증 성공
4. 대시보드 이동

## Flow B. 신규 작업 등록
1. `새 작업` 버튼 클릭
2. 작업 기본 정보 입력
3. 저장
4. 프로젝트 상세 화면 또는 목록으로 이동

## Flow C. 진행 상태 업데이트
1. 작업 상세 화면 진입
2. 단계별 상태 변경
3. 저장 또는 즉시 반영
4. 진행률/상태 배지 갱신

## Flow D. 제출 완료 처리
1. 작업 상세에서 `제출 완료` 체크
2. 제출일 입력 또는 자동 기록
3. 상태 갱신
4. 아카이브 반영

## Flow E. 과거 작업 탐색
1. 아카이브 페이지 진입
2. 검색/필터 적용
3. 작업 선택
4. 상세 열람

## 3. 모바일 와이어프레임 원칙

- 상단 고정 헤더
- 필터는 드로어/바텀시트
- 카드 1열
- 상세 페이지는 섹션 접기/펼치기 허용
- 우측 하단 FAB로 `새 작업` 제공 가능

---

## 4. 상태 시각화 규칙

| 상태 | 표현 |
|---|---|
| 예정 | 회색 배지 |
| 진행 중 | 파랑 또는 보라 배지 |
| 제출 완료 | 초록 배지 |
| 보류 | 노랑/주황 배지 |
| 마감 지남 | 빨강 강조 |

단계별 상태:
- 시작 전: 얇은 테두리
- 진행 중: 채워진 배지
- 완료: 체크 아이콘 포함

---

## 5. UX 메모

- 작업 생성은 1분 이내 입력 가능해야 함
- 날짜 입력은 캘린더 팝오버 지원
- 마감 임박은 D-day로 노출
- 제출 완료 작업은 시각적으로 구분
- 빈 상태 화면도 설계 필요

---

## 6. 빈 상태 예시
