# PRD

## 1. 제품명
Songwriter Archive App  
작사가 작업 관리 및 아카이브 웹앱

## 2. 목적

작사가가 의뢰를 받았을 때 작업별로 다음을 안정적으로 기록하고 관리할 수 있도록 한다.

- 받은 날
- 제출 마감일
- 작업 진행 상태
- 제출 완료 여부
- 과거 작업 이력

현재 문제는 작업이 메신저, 노트, 파일명, 기억에 흩어져 관리되기 쉽다는 점이다.  
이 앱은 작업의 **진행 관리 툴**이면서 동시에 **포트폴리오형 아카이브** 역할을 수행한다.

## 3. 핵심 사용자

### 1차 사용자
- 외주/의뢰 기반으로 작업하는 개인 작사가

### 확장 사용자
- 작곡가
- 번안가
- 콘텐츠 라이터
- 프로젝트 단위 프리랜서

## 4. 핵심 문제

### 사용자 문제
- 어떤 곡이 언제 들어왔는지 기억이 흐려짐
- 마감일이 겹치면 누락 위험이 생김
- 작업이 어디까지 진행됐는지 즉시 파악하기 어려움
- 제출 완료한 작업을 나중에 다시 찾기 번거로움

### 해결 방식
- 작업 1건 = 카드 1개로 표준화
- 단계별 상태를 구조화
- 마감 우선순위를 자동으로 시각화
- 완료 작업은 아카이브로 분리
- 검색/필터/태그로 재탐색 가능하게 설계

## 5. 성공 기준

### 제품 지표
- 사용자가 1분 내 신규 작업을 등록할 수 있다
- 진행 중 작업의 현재 상태를 5초 내 파악할 수 있다
- 지난 작업을 검색 또는 필터로 10초 내 찾을 수 있다
- 마감 임박 작업이 홈에서 즉시 노출된다

### 정성 목표
- 사용자가 “작업실 기록장이 생겼다”는 느낌을 받는다
- 단순 투두앱이 아니라 창작 워크플로우에 맞는 도구로 인식된다

## 6. 핵심 기능 범위

## 6.1 작업 등록
사용자는 아래 정보를 입력한다.

- 곡명 / 가제
- 아티스트명 또는 프로젝트명
- 의뢰처
- 작업 유형
- 받은 날
- 제출 마감일
- 우선순위
- 메모

## 6.2 진행 상태 관리
각 작업은 3개 단계로 관리한다.

- 음절
- 작사 1 - 코러스
- 작사 2 - 나머지

각 단계 상태값:
- `not_started`
- `in_progress`
- `done`

시스템은 각 단계 상태를 바탕으로 전체 진행률을 계산한다.

## 6.3 제출 완료 처리
사용자가 제출 완료 체크를 하면 시스템은 다음을 수행한다.

- 제출 완료 상태 갱신
- 실제 제출일 기록
- 아카이브 조건 충족 시 아카이브에 반영

## 6.4 대시보드
홈 화면에서 아래 정보를 보여준다.

- 마감 임박 작업
- 진행 중 작업 수
- 제출 대기 작업 수
- 이번 달 제출 완료 수
- 최근 완료 작업

## 6.5 아카이브
완료된 작업을 필터링/검색해 다시 볼 수 있게 한다.

기본 필터:
- 연도
- 아티스트
- 작업 유형
- 제출 완료 여부

## 7. 비기능 요구사항

### 성능
- 대시보드 첫 렌더는 체감상 2초 이내
- 목록 필터링은 즉시 반영
- 모바일에서도 사용 가능해야 함

### 보안
- 사용자별 데이터 분리
- Supabase RLS 필수
- 인증되지 않은 사용자는 데이터 접근 불가

### 안정성
- 입력 검증
- 날짜 오류 방지
- 중복 제출 상태 변경 방어
